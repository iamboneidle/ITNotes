# 1. Цель
Docker должен решить задачу: у тебя 10 контейнеров на основе одного образа Ubuntu. Копировать образ 10 раз — расточительно. Нужно, чтобы контейнеры **делили общие данные**, но при этом **не мешали друг другу** при записи.

Решение — **OverlayFS**, файловая система, которая умеет "склеивать" несколько директорий в одну видимую.
___
# 2. Устройство слоев образа
Dockerfile компилируется в стопку **read-only слоёв**. Каждая инструкция — потенциально новый слой:
```Dockerfile
FROM ubuntu:22.04        # слой 1 — базовый образ
RUN apt install nginx    # слой 2 — добавились файлы nginx
COPY ./app /app          # слой 3 — добавились твои файлы
RUN chmod +x /app/run.sh # слой 4 — изменились права
```
Слои **неизменяемы** после сборки. Они хранятся на диске и переиспользуются между образами.
```
┌─────────────────────────┐
│   слой 4: chmod         │  read-only
├─────────────────────────┤
│   слой 3: COPY ./app    │  read-only
├─────────────────────────┤
│   слой 2: apt nginx     │  read-only
├─────────────────────────┤
│   слой 1: ubuntu:22.04  │  read-only
└─────────────────────────┘
```

> [!note] 
> Переиспользование слоёв Если два образа начинаются с `FROM ubuntu:22.04` — первый слой у них **общий на диске**. `docker pull` не скачивает слои, которые уже есть локально.

# 3. Как работает OverlayFS
OverlayFS оперирует четырьмя директориями:

| Директория | Роль                                           |
| ---------- | ---------------------------------------------- |
| `lowerdir` | read-only слои (один или несколько, через `:`) |
| `upperdir` | read-write слой — сюда идут все изменения      |
| `workdir`  | служебная, нужна ядру для атомарных операций   |
| `merged`   | то, что видит процесс в контейнере             |
Как Docker монтирует OverlayFS "под капотом":
```bash
mount -t overlay overlay \
  -o lowerdir=/layer1:/layer2:/layer3,\
     upperdir=/containers/abc/diff,\
     workdir=/containers/abc/work \
  /containers/abc/merged
```
Процесс в контейнере видит только `/merged` — единое дерево файлов.
___
# 4. Copy-on-Write (CoW)
Ключевой принцип: файлы из `lowerdir` **не копируются заранее**. Копирование происходит только в момент **первой записи**.
### Сценарии:
**Чтение файла** — файл берётся прямо из `lowerdir`, без копирования. Быстро и дёшево.
**Запись в существующий файл** — OverlayFS копирует файл из `lowerdir` в `upperdir`, и уже там вносит изменения. В `lowerdir` файл остаётся нетронутым.
**Создание нового файла** — файл сразу создаётся в `upperdir`.
**Удаление файла** — в `upperdir` создаётся специальный файл-маркер (`whiteout`). OverlayFS скрывает оригинал из `lowerdir`, не трогая его физически.
```
lowerdir:  /etc/nginx/nginx.conf  ← оригинал, read-only
                    │
         [первое изменение]
                    ↓
upperdir:  /etc/nginx/nginx.conf  ← копия + изменения
           (lowerdir-версия теперь скрыта)
```

# 5. Расположение на диске
```bash
# Посмотреть реальное расположение слоёв
docker inspect <container_id> | grep -A 10 GraphDriver

# Обычно:
/var/lib/docker/overlay2/
├── <hash>/          # каждый слой
│   ├── diff/        # файлы этого слоя
│   ├── link         # короткий алиас
│   ├── lower        # ссылка на нижние слои
│   └── work/        # служебная директория
└── <hash>/
    └── merged/      # примонтированная файловая система живого контейнера
```

# 6. Ловушка удаления файла
*Большой файл → изменил 1 байт → CoW скопировал весь файл*
Если в контейнере изменить большой файл (например, базу данных прямо в контейнерном слое) — весь файл скопируется в `upperdir`. Поэтому базы данных и всё, куда активно пишется, нужно выносить в **volumes** — они монтируются в обход OverlayFS.

*Удаление файла не освобождает место в образе*
Классическая ловушка:
```Dockerfile
RUN apt install build-essential   # слой раздулся
RUN apt remove build-essential    # whiteout в новом слое,
                                   # но предыдущий слой никуда не делся!
```
Правильно — объединять в одну инструкцию `RUN`:
```Dockerfile
RUN apt install build-essential \
    && make ... \
    && apt remove build-essential \
    && apt clean
```
