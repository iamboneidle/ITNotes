# 1. Определения
**Хардлинк** - жесткая ссылка. По своей сути является тем же файлом на который ссылается. Также счетчик в айнодах есть. Цифра 1 это оно. Счетчик имен одного и того же файла.
Создается командой `ln`.

**Симлинк** - магкая ссылка. При ее создании создается новый объект на существующий файл файловой системы. Отличить можно по наличию буквы l в выводе команды `ls`.
Создается командой `ln -s`.

# 2. Ключевые отличия
- `hardlink` не может указывать на файл в другой файловой системе (так как `inode` может принадлежать только одной ФС), а `symlink` – может.
- при редактировании файла-ссылки в случае с `hardlink`ом – изменятся оба файла, так как это один и тот же объект, а в случае с `symlink`'а – можно изменять его имя, атрибуты, направить его на другой файл и при этом оригинальный файл не будет затронут (но учтите, что если вы откроете файл симлинка для редактирования – то измените оригинальный файл, т.к. по сути вы откроете для редактирования именно его)
- жёсткая ссылка не может указывать на каталог
- При удалении `hardlink`'а – файл будет существовать до тех пор, пока есть хотя бы 1 `hardlink` на него, но может “менять каталог размещения”, если был удалён “исходный” файл, но остался файл-`hardlink` в другом месте. При удалении же файла, на который указывает `symlink`  – файл-ссылка просто станет нерабочим.
# 3. Отличие простыми словами
**1. Жесткая ссылка (Hard link)**
- **Суть:** Это _другое имя_ для того же самого файла (один и тот же inode).
- **Ограничения:** Не работает для папок (директорий) и между разными дисками/разделами.
- **Поведение:** Удаление оригинала (первого имени) **не удаляет** данные, пока есть хотя бы одна жесткая ссылка.
- **Внешний вид:** Выглядит как обычный файл (не отличить по типу).

**2. Символическая ссылка (Symlink / Soft link)**
- **Суть:** Это _отдельный маленький файл_, который содержит **путь** к другому файлу или папке.
- **Ограничения:** Может ссылаться на что угодно (папки, файлы на других дисках, несуществующие файлы).
- **Поведение:** Если удалить оригинал — ссылка «ломается» (висят в никуда). Если удалить ссылку — оригинал не пострадает.
- **Внешний вид:** Помечается специальным атрибутом (`l` в `ls -l`) и стрелкой (`file -> target`).
