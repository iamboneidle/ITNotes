# 2. df — disk free
> Показывает использование **разделов и файловых систем** целиком.

Основные флаги
```bash
df -h          # human-readable (KB/MB/GB)
df -H          # то же, но в степенях 1000, а не 1024
df -T          # показать тип ФС (ext4, tmpfs, ...)
df -i          # показать inode вместо байт
df -a          # включить "виртуальные" ФС (proc, sysfs...)
df --total     # добавить строку с суммой
```
Примеры
```bash
df -h /home             # только раздел, где лежит /home
df -hT | grep -v tmpfs  # исключить tmpfs из вывода
df -h --output=source,size,used,avail,pcent,target
```
Что означают колонки

|Колонка|Смысл|
|---|---|
|`Filesystem`|устройство / раздел|
|`Size`|полный размер|
|`Used`|занято|
|`Avail`|доступно (не то же, что Size − Used, т.к. 5% резерв)|
|`Use%`|процент занятости|
|`Mounted on`|точка монтирования|
> [!warning] 
> 5% резерв ext4 по умолчанию резервирует ~5% для root. Поэтому `Used + Avail ≠ Size`. Изменить: `tune2fs -m 1 /dev/sdX` 

> [!tip] 
>  Диск как будто полон, но df показывает место? Проверь inode: `df -i`. Если inode на 100% — файлов слишком много (типично для mail-серверов, npm, pip-кэшей).

# 3. du — disk usage
>Показывает использование места **конкретными файлами и папками**.

Основные флаги:
```bash
du -h          # human-readable
du -s          # summary — только итог по каждому аргументу
du -a          # показать каждый файл, не только папки
du -c          # добавить строку grand total
du -d 1        # глубина рекурсии (аналог --max-depth=1)
du --exclude=PATTERN   # исключить по маске
du -x          # не переходить на другие ФС (важно при / )
du -L          # следовать symlink'ам
```
Примеры:
```bash
# Размер каждой папки в текущей директории
du -sh *
# То же, отсортировать по убыванию
du -sh * | sort -rh
# Топ-20 самых тяжёлых папок/файлов на всём диске
du -ah / 2>/dev/null | sort -rh | head -20
# Размер конкретной папки
du -sh /var/log
# Только первый уровень вложенности (не рекурсивно)
du -h --max-depth=1 /var
# Что занимает место в /home, не уходя на другие разделы
du -xsh /home/*
# Найти папки тяжелее 1 ГБ
du -h / 2>/dev/null | grep '^[0-9\.]*G'
```

> [!note] 
> du vs df расхождение `du /` и `df /` могут показывать разные цифры. Причина: удалённые файлы, которые ещё держат открытые процессы. Найти их: `lsof | grep deleted`

# 3. ncdu — NCurses Disk Usage
>Интерактивный TUI-браузер дискового пространства. **Лучший инструмент для визуального анализа.**

> [!info] 
> ncdu2 Существует переписанная версия `ncdu2` на Zig — быстрее на больших ФС. Доступна как бинарник на GitHub.

# 4. find — поиск тяжелых файлов
Примеры:
```bash
# Файлы тяжелее 100 МБ
find / -type f -size +100M 2>/dev/null
# Файлы тяжелее 1 ГБ, вывести размер
find / -type f -size +1G -exec ls -lh {} \; 2>/dev/null
# Файлы крупнее 500 МБ в /var, отсортировать
find /var -type f -size +500M -printf '%s %p\n' 2>/dev/null | sort -rn
# Старые логи (не менялись > 30 дней)
find /var/log -type f -mtime +30
# Файлы без владельца (orphaned)
find / -nouser -o -nogroup 2>/dev/null
```

# 5. lsof — удалённые, но открытые файлы
>Когда `df` говорит "диск полон", а `du /` показывает мало — дело в **удалённых файлах, которые держат процессы**.

Примеры:
```bash
# Найти удалённые файлы, занимающие место
lsof | grep '(deleted)'
# Только большие (отсортировать по SIZE)
lsof | grep deleted | awk '{print $7, $1, $2, $9}' | sort -rn | head -20
# Освободить место без перезапуска процесса (обнулить через /proc)
# Узнать PID и FD из lsof, потом:
> /proc/<PID>/fd/<FD>
```

# 6. Доп инструменты
### iostat — активность I/O
```bash
iostat -xh 1        # расширенная статистика, обновление каждую секунду
iostat -d /dev/sda  # только конкретный диск
```
### lsblk — структура блочных устройств
```bash
lsblk              # дерево дисков, разделов, LVM
lsblk -f           # + тип ФС, UUID, метка
lsblk -o NAME,SIZE,FSUSED,FSAVAIL,MOUNTPOINT
```
### fdisk / parted — таблица разделов
```bash
sudo fdisk -l           # показать все диски и разделы
sudo parted -l          # альтернатива, понимает GPT
```
### stat — метаданные файла
```bash
stat файл.txt          # размер, inode, время доступа/изменения
```
### ls с сортировкой по размеру
```bash
ls -lhS            # сортировка по размеру (по убыванию)
ls -lhSr           # по возрастанию
```
### Места, которые часто забывают проверить

```bash
# Журналы systemd
journalctl --disk-usage
sudo journalctl --vacuum-size=500M   # обрезать до 500 МБ

# Кэш apt
du -sh /var/cache/apt
sudo apt clean

# Docker
docker system df
docker system prune

# Корзина
du -sh ~/.local/share/Trash/

# Временные файлы
du -sh /tmp /var/tmp

# Snap пакеты (старые версии)
df -h /var/lib/snapd/snaps
snap list --all

# Coredumps
ls -lh /var/lib/systemd/coredump/
sudo coredumpctl list
```

# 7. Типичный алгоритм расследования
```bash
1. df -h          → на каком разделе проблема?
2. df -i          → не забиты ли inode?
3. lsof | grep deleted → нет ли удалённых, но открытых файлов?
4. sudo ncdu -x / → интерактивно найти виновника
5. du -sh /var /home /tmp /opt → быстро сравнить крупные папки
6. journalctl --disk-usage → проверить логи systemd
7. docker system df → если используется Docker
```
