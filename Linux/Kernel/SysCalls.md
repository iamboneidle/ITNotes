# 1. Системные вызовы
**Системный вызов** — это то, посредством чего код приложения, выполняющегося в пользовательском режиме, запрашивает службу, предоставляемую кодом, который выполняется в режиме ядра.

Команда для вывода всех системных вызовов во время исполнения программы:
```shell
strace -c ls
```

Условный топ-10:
1. **`read`** — чтение данных из файлового дескриптора
2. **`write`** — запись данных в файловый дескриптор
3. **`open`** — открытие файла (возвращает дескриптор)
4. **`close`** — закрытие файлового дескриптора
5. **`fork`** — создание нового процесса (копия текущего)
6. **`execve`** — замена текущего процесса новой программой
7. **`wait` / `waitpid`** — ожидание завершения дочернего процесса
8. **`socket`** — создание сетевого сокета
9. **`mmap`** — отображение файла или устройства в память
10. **`ioctl`** — управление устройством (специфичные операции)

Посмотреть информацию по вызовам можно командой `man 2 <syscall>` из пакета `man-pages`.
