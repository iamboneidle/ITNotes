# 1. Масштабирование контроллера репликации
Данная команда позволяет изменять количество реплик модуля в контроллере репликации:
```shell
kubectl scale rc kubia --replicas=10
```
## 1.1. Масштабирование контроллера репликации путем изменения его определения
```shell
kubectl edit rc kubia
```
Далее откроется текстовый редактор.
```yaml
apiVersion: v1
kind: ReplicationController
metadata:
...
spec:
  replicas: 3 #<- тут можно менять количество реплик
  selector:
    app: kubia
...
```
___
# 2. Декларативный подход к масштабированию
Все что надо сделать - это сказать k8s, сколько модулей требуется. Ничего больше делать не нужно, далее всю работу кластер сделает сам.
___
# 3. Удаление контроллера репликации
При удалении контроллера репликации с помощью команды `kubectl delete` модули также удаляются. Но поскольку модули, созданные контроллером репликации, не являются неотъемлемой частью контроллера репликации и лишь им управляются, можно удалить только контроллер репликации и оставить модули работающими.

При удалении контроллера репликации с помощью команды `kubectl delete`
его модули можно сохранить, передав команде параметр `--cascade=false`:
```shell
kubectl delete rc kubia --cascade=false
```
