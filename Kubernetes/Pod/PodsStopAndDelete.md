# 1. Удаление модуля
## 1.1. Удаление модуля по имени
```shell
kubectl delete pod <pod_name>
```
Удаляя модуль, вы поручаете Kubernetes завершить работу всех контейнеров,
входящих в этот модуль. Kubernetes посылает сигнал `SIGTERM` процессу и ожидает определенное количество секунд (по умолчанию 30) для корректного
завершения работы. 
Если он не выключается вовремя, процесс затем убивается через `SIGKILL`. Для того чтобы убедиться, что ваши процессы всегда корректно
завершают работу, они должны правильно обрабатывать сигнал SIGTERM.

> [!tip] 
> Можно также удалить несколько модулей, указав несколько имен, разде-
ленных пробелами (например, `kubectl delete pod pod1 pod2`). 
## 1.2. Удаление модулей с помощью меток
Оба модуля содержат метку `creation_method=manual`, поэтому их можно удалить с помощью селектора меток:
```shell
kubectl delete po -l creation_method=manual
```
## 1.3. Удаление модулей путем удаления всего пространства имен
Вы можете удалить все пространство имен (модули будут удалены автоматически вместе с пространством имен), используя следующую команду:
```shell
kubectl delete ns custom-namespace
```
## 1.4. Удаление всех модулей в пространстве имен, не удаляя само пространство имен
Можно удалить все модули в пространстве имен с помощью флага `-all`:
```shell
kubectl delete po --all
```
## 1.5. Удаление (почти) всех ресурсов в пространстве имен
```
kubectl delete all --all
```
Первый параметр all в данной команде указывает, что вы удаляете ресурсы
всех типов, а параметр --all указывает, что вы удаляете все экземпляры ресурсов, вместо того чтобы указывать их по имени.

> [!hint] 
> Перейти в другое пространство имен по умолчанию для текущего контекста можно командой `kubectl config set-context --current --namespace=my-namespace`.
