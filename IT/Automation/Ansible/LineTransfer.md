# 1. Синтаксис и виды переносов строк
В YAML-синтаксисе есть несколько способов многострочной записи:
## 1.1. Pipe
### 1.1.1. | (Pipe) - Сохраняет переносы строк
Код:
```yaml
text: |
  Первая строка
  Вторая строка
  Третья строка
```
Результат:
```
Первая строка
Вторая строка
Третья строка
```
### 1.1.2. |- (Pipe with strip) - Убирает конечные переносы
Код:
```yaml
text: |-
  Первая строка
  Вторая строка
  Третья строка
  
```
Результат:
```
Первая строка
Вторая строка
Третья строка <- No new line
```
### 1.1.3. |+ (Pipe with keep) - Сохраняет все конечные переносы
Код:
```yaml
text: |+
  Первая строка
  Вторая строка
  Третья строка
  
```
Результат:
```
Первая строка
Вторая строка
Третья строка 
		     <- New line
```
## 1.2. Fold
### 1.2.1. > (Fold) - Сворачивает строки в пробелы
Код:
```yaml
text: >
  Первая строка
  Вторая строка
  Третья строка
```
Результат
```
Первая строка Вторая строка Третья строка
```
### 1.2.2. >- (Fold with strip) - Убирает конечные переносы
Код:
```yaml
text: >-
  Первая строка
  Вторая строка
  Третья строка
  
```
Результат
```
Первая строка Вторая строка Третья строка <- No new line
```
### 1.2.3. >+ (Fold with keep) - Сохраняет пустые строки внутри текста
Код:
```yaml
text: >+
  Первая строка
  Вторая строка
  
  
  Третья строка
```
Результат
```
Первая строка Вторая строка


Третья строка
```

# 2. Кавычки
## 2.1. Двойные кавычки - Экранирование спецсимволов
```yaml
text: "Первая строка\nВторая строка\tТабуляция"
```
Нужно экранировать спецсимволы: `\n`, `\t`,`"`, `\`.
## 2.2. Одинарные кавычки - Без интерполяции
```yaml
text: 'Текст как есть $VAR \n не работает'
```
Всё внутри считается литералом, даже спецсимволы.
