# Вопросы
#### 1. Что такое IaaC и зачем он нужен?
> Подход к управлению и настройке ИТ-инфраструктуры с помощью **файлов конфигурации** (кода), а не через ручную настройку серверов.
>- Автоматизация и скорость
>- Масштабируемость
>- Версионность
>- Снижение рисков: Минимизируется человеческий фактор
#### 2. Что такое Terraform?
> **Terraform** — это Open Source инструмент для **IaC (Infrastructure as Code)** от компании HashiCorp, который позволяет создавать, изменять и версионировать инфраструктуру безопасно и эффективно.
> 
> Основные фишки:
> - **Декларативный подход:** Ты описываешь в конфигах (`.tf`), **что** ты хочешь получить (например, «нужно 3 сервера»), а Terraform сам вычисляет, **как** это сделать.
> - **Провайдеры:** Terraform умеет работать практически с чем угодно через API — от облаков (AWS, Azure, GCP) до локальных решений (VMware, OpenStack) и даже SaaS (Cloudflare, Datadog).
> - **State File (`terraform.tfstate`):** Инструмент хранит текущее состояние инфраструктуры в специальном файле. Это позволяет ему понимать, какие ресурсы уже созданы, а какие нужно обновить или удалить.
> - **Execution Plan:** Перед применением изменений команда `terraform plan` показывает точный список действий, которые будут выполнены.
В отличие от того же Ansible (который лучше подходит для настройки софта _внутри_ ОС), Terraform идеален для создания **фундамента**: сетей, виртуалок, балансировщиков и баз данных.
#### 3. Какие инструменты автоматизации вы знаете?
> **Управление инфраструктурой (IaC):**
> - Terraform/OpenTofu;
> - CLoudFormation.
> **Управление конфигурациями:**
> - Ansible;
> - Helm;
> - Chef/Puppet.
> **CI/CD:**
> - Jenkins;
> - GitLab CI;
> - GitHub Actions.
