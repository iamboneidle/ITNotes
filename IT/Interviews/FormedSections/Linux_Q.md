# Вопросы
#### 1. cat file1 > file2 & cat file1 >> file2
>- `cat file1 > file2` перепишет `file2` содержанием `file1`;
>- `cat file1 >> file2` допишет к имеющемуся содержимому `file2` содержимое `file1`.
#### 2. ${hostname} & $(hostname)
>- `${hostname}` — это подстановка **переменной оболочки** (shell variable) с именем `hostname`. Если переменной нет, подставится пустая строка.
>- `$(hostname)` — это **подстановка команды** (command substitution).
#### 3. Как система понимает, где искать команду curl
>1. Проверка на наличие в списке встроенных команд самого `shell` по типу `cd`, `echo` и т.д.
>2. Проверка директорий, перечисленных в `$PATH` типо `/usr/bin/local`, `/usr/bin` и т.д. Shell последовательно будет искать исполняемый файл в этих директориях и при первом вхождении остановит поиск и выполнит первый попавшийся.
>3. Если не найден — выдаст ошибку.
#### 4. Опишите общую архитектуру операционной системы
>Снизу вверх:
>1. Аппаратура (процессор, память и т.д.);
>2. Ядро (управление процессами, памятью, сеть и т.д.);
>3. Системные вызовы (`kernel space` <—> `user space`);
>4. Пользовательские библиотеки типо `libc`, `glibc`;
>5. Системные утилиты, демоны типо `systemd`, `cron`;
>6. Прикладное ПО.
#### 5. Опишите основное предназначение операционной системы
>Быть посредником м/у hardware и пользователем. Исполняет роли:
>- Управляет ресурсами;
>- Является интерфейсом взаимодействия с железом для пользователя;
>- Защищает и изолирует программы друг от друга, защищает систему от вредоносных действий.
#### 6. Зачем нужны файловые системы? Какие существуют?
>Чтобы **организовать хранение данных** на диске: давать файлам имена, раскладывать их по папкам, отслеживать, где на диске что лежит, и управлять доступом.
>**Linux**:
>- `ext4` — стандартная для большинства дистрибутивов;
>- `ext2/ext3` — старые версии;
>- `btrfs`, `XFS` — для продвинутых сценариев (снапшоты, большие объёмы).
>**Windows**:
>- `NTFS` — основная (журналирование, права, большие файлы);
>- `FAT32` — старая, совместимая, но файлы ≤4GB.
>**Сетевые и распределенные**:
>- `NFS`, `SMB (CIFS)` — сетевые диски;
>- `HDFS` — для Hadoop.
#### 7. В чем разница между виртуализацией и контейнеризацией? В чем преимущества контейнеров?
>**Виртуализация** — каждая виртуальная машина содержит **полноценную гостовую ОС** со своим ядром. Гипервизор эмулирует аппаратуру. ВМ тяжелые (гигабайты), запускаются минутами, но изоляция максимальная.
>**Контейнеризация** — контейнеры используют **общее ядро хостовой ОС**, изолируясь через пространства имен (namespaces). Контейнеры легкие (мегабайты), запускаются за секунды, работают почти без потери производительности.
>Преимущества:
>- **Легковесность** — меньше занимают места и ресурсов
>- **Быстрый запуск** — миллисекунды/секунды вместо минут
>- **Портативность** — работают одинаково на любой системе с поддержкой контейнеров (локально, в облаке, на сервере)
>- **Удобство CI/CD** — легко автоматизировать сборку, тестирование, деплой
>- **Масштабируемость** — легко запустить десятки копий одного контейнера
>- **Воспроизводимость** — образ содержит всё необходимое (код + зависимости)
#### 8. Какова файловая структура в Linux (UNIX) системах, расположенных в /etc, /dev, /proc, /sys, /lib, /var (несколько директорий на выбор)?
> [[FileSystemBase]]
>[[PseudoFileSystem]]
>- `/etc` — конфигурационные файлы системы (настройки программ, сети, пользователей). Примеры: `passwd`, `fstab`, `nginx/`.
>- `/dev` — файлы устройств. Диски (`/dev/sda`), терминалы (`/dev/tty`), случайные числа (`/dev/random`). Это не настоящие файлы, а интерфейсы для общения с драйверами.
>- `/proc` — виртуальная ФС, содержащая информацию о **процессах и ядре**. Например, `/proc/cpuinfo`, `/proc/meminfo`, `/proc/[PID]/`. Данные в реальном времени.
>- `/sys` — виртуальная ФС для информации об **устройствах и драйверах** ядра (более структурированная, чем `/proc`). Используется для управления через sysfs.
>- `/lib` — системные библиотеки (как `.dll` в Windows) и модули ядра, необходимые для работы программ в `/bin` и `/sbin`.
>- `/var` — изменяемые данные: логи (`/var/log`), очереди печати (`/var/spool`), временные файлы (`/var/tmp`), кэш пакетов (`/var/cache`).

#### 13. С помощью чего можно собрать информацию о текущем состоянии процессора, памяти, диска, сети?
>[[UsefulCommands]]
>[[SpaceLeftInspection]]
>**Чтение виртуальных файлов** `procfs` & `sysfs`:
>- Процессор: `/proc/cpuinfo`;
>- Память: `/proc/meminfo`;
>- Диск: `/proc/diskstats`, `/proc/partitions`, `/proc/mounts`;
>- Сеть: `/proc/net/dev`.
>**Утилиты**:
>- Процессор: `top`/`htop`, `mpstat`, `vmstat`;
>- Память: `free -h`, `vmstat`, `smem`;
>- Диск: `df -h`, `iotop`, `iostat`, `du`;
>- Сеть: `ip -s link`, `ss -s`, `nload`/`iftop`.

#### 18. В чем разница между Ctrl+C и Ctrl+Z?
>[[Signals]]
>- `Ctrl+C` — **прерывает (завершает)** выполняющийся процесс. Отправляет сигнал `SIGINT` (сигнал прерывания).
>- `Ctrl+Z` — **приостанавливает (ставит на паузу)** выполняющийся процесс. Отправляет сигнал `SIGTSTP`. Процесс **остается в памяти**, но не выполняется. Вернуться к нему можно командой `fg` (foreground — запустить на переднем плане) или `bg` (background — запустить в фоне).
#### 19. Как перенаправить одновременно stderr и stdin?
>- `command < input_file.txt 2> error_file.txt` — данные читаются из `input_file.txt`, а ошибки записываются в `error_file.txt`;
>- `command < input_file.txt > output_and_errors.txt 2>&1` — ввод из файла, а все выводы (обычный и ошибки) идут в один файл.
#### 20. Как убить процесс? Какие есть типы сигналов?

> `SIGTERM` — мягко;

#### 21. Что делает команда grep?
>`grep` (Global Regular Expression Print) — ищет строки, соответствующие заданному шаблону (обычно регулярному выражению), и выводит их.
#### 22. Что такое скрипт bash?
>**Bash-скрипт** — это текстовый файл, содержащий последовательность команд, которые выполняются интерпретатором **bash** (Bourne Again SHell) одна за другой, как если бы вы вводили их в терминале вручную.
