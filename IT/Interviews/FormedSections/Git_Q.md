# Вопросы
#### 1. Как решить merge conflict? Что такое rebase, cherry-pick?
>Конфликт решается ручным выбором нужного кода между маркерами `HEAD` и входящей ветки. **Rebase** — это переписывание истории для создания линейного графа (перенос базы ветки). **Cherry-pick** — это точечный перенос одного конкретного коммита из одной ветки в другую
#### 2. В чем разница между git merge и git rebase?
>**Merge** создает новый коммит слияния, сохраняя историю в виде графа. **Rebase** переносит твои коммиты в конец целевой ветки, создавая линейную историю.
#### 3. Какая разница между Git pull/Git fetch?
>**Fetch** только скачивает изменения в локальную копию репозитория, не меняя рабочие файлы. **Pull** — это Fetch плюс автоматическое слияние (Merge) скачанных изменений в твою текущую ветку.
#### 4. Версионирование. Какая разница между SemVer и CalVer?
>**SemVer** — Это стандарт индустрии для библиотек и API. Формат: **MAJOR.MINOR.PATCH** (например, `2.3.1`).
>**CalVer** — Версия базируется на дате релиза. Формат может быть разным: **YYYY.MM.Micro** (например, `2024.04.2`) или **YY.Minor**.
>**Примеры:** Ubuntu (`24.04`), PyCharm (`2023.3.1`), Terrafom (иногда использует смешанные подходы).
