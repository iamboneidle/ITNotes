# Вопросы
#### 1. Что такое DevOps?
>- **DevOps** — подход, объединяющий разработку и эксплуатацию для автоматизации и ускорения доставки кода в продакшен.
>- *Цель* — быстрее выпускать релизы с меньшим количеством багов.
#### 2. Объясните концепцию Infrastructure as Code, зачем это нужно и какие проблемы решает?
>**Infrastructure as Code (IaC)** — управление инфраструктурой (серверы, сети, балансировщики, диски) через конфигурационные файлы, которые хранятся в системе контроля версий, вместо ручных изменений через консоль или интерфейс облачного провайдера.
>*Цель*:
>- **Воспроизводимость** — поднять одинаковую среду (dev/stage/prod) одной командой.
>- **Скорость** — развернуть 10 серверов за минуту, а не за час.
>- **Документация** — конфиги сами описывают состояние инфраструктуры.
>- **Контроль версий** — любое изменение логируется, можно откатиться.
#### 3. Что такое cookies? Зачем нужны? JWT?
>**Cookies** — это небольшие фрагменты текстовых данных, которые сервер отправляет в браузер пользователя. Браузер сохраняет их у себя и автоматически прикрепляет к каждому последующему запросу к этому же серверу.
>- Управление сессиями
>- Персонализация
>- Трекинг
>
>**JWT** — это открытый стандарт (RFC 7519) для безопасной передачи информации между сторонами в виде JSON-объекта. В отличие от сессионных кук, JWT содержит в себе всю необходимую информацию о пользователе, поэтому серверу не нужно лезть в базу данных или кэш (Redis), чтобы проверить валидность токена.
>- Header
>- Payload
>- Signature
#### 4. Что такое feature toggles и зачем они?
>**Feature Toggles** (также известные как Feature Flags) — это техника разработки ПО, которая позволяет изменять поведение системы во время выполнения (runtime) без изменения кода или развертывания новой версии.
#### 5. Что такое TDD (Test Driven Development) и BDD (Behaviour Driven Development)?
>**TDD (Test-Driven Development)**
>**Разработка через тестирование.** Основная идея: сначала пишется тест для функции, которая еще не существует, а затем — минимальный код, чтобы это
>
>**BDD (Behaviour-Driven Development)**
>**Разработка через поведение.** Это расширение TDD, которое фокусируется на взаимодействии между разработчиками, тестировщиками и бизнесом (заказчиком).
>Вместо проверки того, как работает конкретный метод в коде (как в TDD), BDD описывает сценарии использования программы на человекочитаемом языке.
#### 6. Что такое SDLC?
>**SDLC (Software Development Life Cycle)** — это жизненный цикл разработки программного обеспечения. Если говорить проще, это четко структурированный план или «дорожная карта», которая описывает каждый этап создания IT-продукта: от возникновения идеи до момента, когда программа устаревает и выводится из эксплуатации.
>1. Анализ требований
>2. Проектирование
>3. Разработка
>4. Тестирование
>5. Развертывание
>6. Поддержка
#### 7. Что такое configuration drift?
>**Configuration Drift** (дрейф конфигурации) — это явление, при котором фактическое состояние ИТ-инфраструктуры (серверов, сетевого оборудования, контейнеров) со временем начинает отклоняться от эталонного или задокументированного состояния.
>
>1. Ручные правки
>2. Несовершенство скриптов (неидемпотентность)
>3. Обновления софта
#### 8. Назовите три важных KPI для DevOps-специалиста.
>1. Deployment Frequency
>2. Change Failure Rate
>3. Mean Time to Recovery
