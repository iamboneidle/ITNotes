# Общие вопросы
#### Что такое DevOps?
>- **DevOps** — подход, объединяющий разработку и эксплуатацию для автоматизации и ускорения доставки кода в продакшен.
>- *Цель* — быстрее выпускать релизы с меньшим количеством багов.
#### Объясните концепцию Infrastructure as Code, зачем это нужно и какие проблемы решает?
>**Infrastructure as Code (IaC)** — управление инфраструктурой (серверы, сети, балансировщики, диски) через конфигурационные файлы, которые хранятся в системе контроля версий, вместо ручных изменений через консоль или интерфейс облачного провайдера.
>*Цель*:
>- **Воспроизводимость** — поднять одинаковую среду (dev/stage/prod) одной командой.
>- **Скорость** — развернуть 10 серверов за минуту, а не за час.
>- **Документация** — конфиги сами описывают состояние инфраструктуры.
>- **Контроль версий** — любое изменение логируется, можно откатиться.
