# 1. ENTRYPOINT
Инструкция, которая задаёт **точку входа** — команду, которая **всегда** выполняется при запуске контейнера. Контейнер ведёт себя как исполняемый файл.
```dockerfile
ENTRYPOINT ["/app/start.sh"]          # exec-форма (рекомендуется)
ENTRYPOINT /app/start.sh              # shell-форма
```
- **exec-форма** — запускает процесс напрямую, без оболочки. Процесс получает PID 1 и корректно обрабатывает сигналы (`SIGTERM`, `SIGINT`).
- **shell-форма** — интерпретируется как `/bin/sh -c "..."`. PID 1 получает `/bin/sh`, а не само приложение — сигналы обрабатываются хуже.

Отменить `ENTRYPOINT` можно только явно через флаг при запуске:
```bash
docker run --entrypoint /bin/bash my_image
```
___
# 2. CMD
Инструкция, которая задаёт **команду или аргументы по умолчанию**. Выполняется только если при запуске контейнера не передана другая команда.
```dockerfile
CMD ["node", "app.js"]           # exec-форма — команда по умолчанию
CMD ["param1", "param2"]         # аргументы для ENTRYPOINT
CMD node app.js                  # shell-форма
```

Если в `docker run` передать любой аргумент — `CMD` **полностью игнорируется**:
```bash
docker run my_image              # выполнится CMD
docker run my_image /bin/bash    # CMD проигнорирован, запустится bash
```
> [!note] 
> Если в Dockerfile несколько инструкций `CMD` — выполняется только **последняя**.

# 3. Сравнение и использование вместе

| |`ENTRYPOINT`|`CMD`|
|---|---|---|
|**Роль** |Исполняемый файл|Аргументы по умолчанию|
|**Переопределение**|Только `--entrypoint` флаг|Любым аргументом в `docker run`|
Когда оба заданы — `CMD` становится **аргументами по умолчанию** для `ENTRYPOINT`, которые можно легко переопределить:
```dockerfile
ENTRYPOINT ["/bin/fortuneloop.sh"]
CMD ["10"]
```
```bash
docker run my_image       # → /bin/fortuneloop.sh 10  (CMD подставился)
docker run my_image 15    # → /bin/fortuneloop.sh 15  (CMD перетёрт)
```

> [!tip] Формула `ENTRYPOINT` = **что запускать**, `CMD` = **с какими аргументами по умолчанию**

> [!warning] Совместимость форм Чтобы `CMD` работал как аргументы для `ENTRYPOINT` — обе инструкции должны быть в **exec-форме**. Shell-форма `ENTRYPOINT` игнорирует `CMD`.
