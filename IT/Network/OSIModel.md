# 7.  Прикладной (Application)
**Что делает:** Интерфейс между пользователем/программой и сетью. Предоставляет сетевые сервисы приложениям.

**Протоколы:**
- `HTTP` / `HTTPS` — веб
- `FTP` / `SFTP` — передача файлов
- `SMTP` / `POP3` / `IMAP` — электронная почта
- `DNS` — разрешение имён
- `DHCP` — динамическое назначение IP
- `SNMP` — управление сетью
- `Telnet` / `SSH` — удалённый доступ
- `LDAP` — каталоги
- `NTP` — синхронизация времени

**Единица данных:** Сообщение (Message)
___
# 6.  Представления (Presentation)

**Что делает:** Преобразование форматов данных, шифрование, сжатие. «Переводчик» между разными системами.

**Протоколы / стандарты:**
- `SSL` / `TLS` — шифрование
- `JPEG`, `PNG`, `GIF` — форматы изображений
- `MPEG`, `MP4` — видео
- `ASCII`, `Unicode` — кодировки текста
- `XDR` — представление данных

**Единица данных:** Сообщение (Message)
___
# 5.  Сеансовый (Session)

**Что делает:** Установка, управление и завершение сеансов связи между приложениями. Синхронизация диалога.

**Протоколы:**
- `NetBIOS` — сессии в сетях Windows
- `RPC` — удалённый вызов процедур
- `PPTP` — туннелирование VPN
- `SAP` — объявление сервисов
- `SIP` — сигнализация VoIP
- `H.245` — мультимедийные сессии

**Единица данных:** Данные (Data)
___
# 4. Транспортный (Transport)

**Что делает:** Надёжная (или ненадёжная) доставка данных между конечными точками. Сегментация, контроль потока, исправление ошибок.

**Протоколы:**
- `TCP` — надёжная передача с установлением соединения
- `UDP` — быстрая передача без гарантий доставки
- `SCTP` — потоковое управление
- `DCCP` — с контролем перегрузки без надёжности

**Ключевые понятия:**
- Порты (0–65535)
- Трёхстороннее рукопожатие TCP (SYN → SYN-ACK → ACK)
- Мультиплексирование

**Единица данных:** Сегмент (TCP) / Датаграмма (UDP)
___
# 3. Сетевой (Network)

**Что делает:** Логическая адресация и маршрутизация пакетов между сетями.

**Протоколы:**
- `IP` (IPv4 / IPv6) — адресация и маршрутизация
- `ICMP` — диагностика (ping, traceroute)
- `IGMP` — управление мультикастом
- `OSPF` — маршрутизация внутри AS
- `BGP` — маршрутизация между AS
- `RIP` — дистанционно-векторная маршрутизация
- `IPsec` — шифрование на сетевом уровне
- `ARP` — разрешение IP → MAC *(иногда относят к L2)*

**Устройства:** Маршрутизатор (Router), L3-коммутатор

**Единица данных:** Пакет (Packet)
___
# 2. Канальный (Data Link)

**Что делает:** Передача кадров между узлами в одной сети. Физическая адресация (MAC), обнаружение ошибок.

**Подуровни:**
- **LLC** (Logical Link Control) — управление логическим каналом
- **MAC** (Media Access Control) — контроль доступа к среде

**Протоколы:**
- `Ethernet` (IEEE 802.3) — проводные LAN
- `Wi-Fi` (IEEE 802.11) — беспроводные LAN
- `PPP` — точка-точка
- `VLAN` (IEEE 802.1Q) — виртуальные сети
- `STP` / `RSTP` — предотвращение петель
- `HDLC` — канальный протокол WAN
- `Frame Relay` — WAN
- `ATM` — асинхронный режим передачи

**Устройства:** Коммутатор (Switch), Мост (Bridge)

**Единица данных:** Кадр (Frame)
___
# 1. Физический (Physical)

**Что делает:** Передача битов по физической среде. Определяет электрические, оптические, механические характеристики.

**Стандарты / технологии:**
- `RS-232` — последовательный интерфейс
- `USB` — универсальная шина
- `DSL` — цифровая абонентская линия
- `SONET` / `SDH` — оптические сети
- `Bluetooth` (физический уровень)
- `IEEE 802.3` (Ethernet физический)
- `IEEE 802.11` (Wi-Fi физический)
- `ISDN` — цифровые телефонные сети
- `OTN` — оптические транспортные сети

**Устройства:** Концентратор (Hub), Репитер, Кабели, NIC

**Единица данных:** Бит (Bit)
