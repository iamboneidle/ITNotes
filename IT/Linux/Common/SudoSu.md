# 1. su
Команда `su` вызывает повторную регистрацию в системе в предопределенной оболочке другого пользователя. По умолчанию это суперпользователь `UID=0`.
Требует ввести имя и пароль того пользователя, чьи полномочия нужно получить.
___
# 2. sudo
Команда `sudo` реализует явные правила передачи полномочий из `/etc/sudoers` и требует подтверждение паролем того пользователя, который команду выполняет, т.е. запрашивает права.

Данные правила из `/etc/sudoers` можно просмотреть/изменить командой:
```shell
sudo visudo
```
