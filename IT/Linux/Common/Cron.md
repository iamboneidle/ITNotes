# 1. Что такое Cron
`cron` — это системный планировщик задач в Unix-подобных системах. Он используется для автоматического выполнения задач (скриптов, программ и команд) по расписанию.
___
# 2. Синтаксис
Формат стандартного cron-выражения (в `crontab -e`):
```scss
┌───────────── минута (0–59)
│ ┌───────────── час (0–23)
│ │ ┌───────────── день месяца (1–31)
│ │ │ ┌───────────── месяц (1–12)
│ │ │ │ ┌───────────── день недели (0–7) (0 и 7 — воскресенье)
│ │ │ │ │
│ │ │ │ │
* * * * * <команда>
```
## 2.1. Поля cron-выражения
| Позиция | Название    | Допустимые значения | Описание                      |
| ------- | ----------- | ------------------- | ----------------------------- |
| 1       | Минута      | 0–59                | В какой минуте часа выполнять |
| 2       | Час         | 0–23                | В каком часу суток выполнять  |
| 3       | День месяца | 1–31                | В какой день месяца выполнять |
| 4       | Месяц       | 1–12 или jan–dec    | В каком месяце выполнять      |
| 5       | День недели | 0–7 или sun–sat     | В какой день недели выполнять |

> [!important] 
> День месяца и день недели работают по логике **ИЛИ**: если одно из условий выполняется, задача будет запущена. 

# 2.2. Спец-символы
- `*` - любое допустимое значение;
- `,` - перечисление значений (например, `1,15,30`);
- `-` - диапазон значений (например, `1-5`);
- `/` - шаг (например, `*/10` — каждые 10 единиц).
Примеры:
- `*/5 * * * *` — каждые 5 минут
- `0 12 * * 1-5` — в 12:00 с понедельника по пятницу
- `15 10 1 * *` — в 10:15 утра 1 числа каждого месяца
## 2.3. Ключевые слова
|Шаблон|Эквивалент|Описание|
|---|---|---|
|`@reboot`|—|Один раз при старте системы|
|`@yearly`|`0 0 1 1 *`|Раз в год|
|`@monthly`|`0 0 1 * *`|Раз в месяц|
|`@weekly`|`0 0 * * 0`|Раз в неделю|
|`@daily`|`0 0 * * *`|Каждый день|
|`@hourly`|`0 * * * *`|Каждый час|
___
# 3. Примеры
- `0 0 * * *` - каждый день в полночь
- `*/30 * * * *` - каждые 30 минут
- `15 14 1 * *` - 1-го числа месяца в 14:15
- `0 22 * * 1-5` - в 22:00 с понедельника по пятницу
- `0 8-18/2 * * *` - каждые 2 часа с 8:00 до 18:00
- `0 12 * * 0,6` - в полдень по субботам и воскресеньям
___
# 4. Хранение заданий
- Пользовательские задания: `crontab -e`
- Для всех пользователей: `/etc/crontab`, `/etc/cron.d/`
- Каталоги:
    - `/etc/cron.hourly/`
    - `/etc/cron.daily/`
    - `/etc/cron.weekly/`
    - `/etc/cron.monthly/`
___
# 5. Полезные команды
###### 5.1. Просмотр списка задач
```shell
crontab -l
```
###### 5.2. Редактирование задач текущего пользователя
```shell
crontab -e
```
###### 5.3. Удаление всех задач пользователя
```shell
crontab -r
```
###### 5.4. Редактирование задач другого пользователя
```shell
sudo crontab -e -u <user>
```
___
# 6. Логи выполнения
В большинстве систем логи находятся в:
`/var/log/syslog` - Debian/Ubuntu
`/var/log/cron` - RHEL/CentOS

systemd:
```shell
journalctl -u cron
```
