# 1. Network Performance
## 1.1. Очереди соединений
- **`net.core.somaxconn`**: Максимальное количество полностью установленных соединений (TCP Established), ожидающих, пока приложение (Nginx, Python) сделает `accept()`.
- **`net.ipv4.tcp_max_syn_backlog`**: Очередь для «полуоткрытых» соединений (те, кто прислал SYN, но еще не завершил Handshake). Защищает от SYN-флуда и помогает при медленном коннекте.
## 1.2. Буферы приема и передачи (TCP Windows)
Чтобы «прокачать» много данных через толстый канал с большой задержкой (BDP — Bandwidth Delay Product), нужны большие буферы.
- **`net.core.rmem_max`** и **`net.core.wmem_max`**: Максимальный размер буферов чтения и записи для всех типов сокетов.
- **`net.ipv4.tcp_rmem`** и **`net.ipv4.tcp_wmem`**: Автоподстройка буферов именно для TCP. Содержит три значения: min, default, max.
## 1.3. Повторное использование портов (Ephemeral Ports)
Если у тебя прокси (Nginx/Envoy), который открывает тысячи соединений к бэкенду, порты могут закончиться.

- **`net.ipv4.ip_local_port_range`**: Расширяем диапазон (например, `1024 65535`).
- **`net.ipv4.tcp_tw_reuse = 1`**: Позволяет ядру быстрее переиспользовать сокеты, находящиеся в состоянии `TIME_WAIT`.

# 2. Memory Performance
## 2.1. Swappiness
[[Swap]]
- **`vm.swappiness`**: Насколько охотно ядро вытесняет анонимную память (память процессов) в SWAP вместо очистки Page Cache (файлового кэша).
## 2.2. Dirty Pages
[[Swap#3. Cash & Dirty pages]]
Когда процесс пишет файл, данные сначала попадают в Page Cache («грязные страницы»).

- **`vm.dirty_ratio`**: Максимальный процент памяти под грязные страницы. При достижении этого порога все процессы блокируются, пока данные не запишутся на диск (вызывает дикие лаги I/O).
- **`vm.dirty_background_ratio`**: Процент, при котором фоновый процесс ядра (`pdflush`/`kworker`) начинает плавно скидывать данные на диск.
    - _Совет:_ На быстрых дисках (NVMe) эти значения лучше уменьшать, чтобы запись шла короткими частыми порциями, а не одним огромным «фризом».
## 2.3. VFS Cache Pressure
**`vm.vfs_cache_pressure`**: Насколько активно ядро должно «вымывать» кэш инодов и директорий (dentry) из памяти.
- _Значение < 100:_ Ядро будет стараться хранить метаданные файлов в памяти (ускоряет работу с миллионами файлов).
