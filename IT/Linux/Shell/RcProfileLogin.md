# 1. Как работает оболочка
**Login shell** — когда ты входишь в систему (SSH, консоль входа). Читает файлы по порядку: `/etc/profile` → `~/.bash_profile` → `~/.bash_login` → `~/.profile`

**Interactive shell** — когда открываешь новый терминал внутри системы. Читает только `~/.bashrc`

Вот что делает каждый файл:

- **`/etc/profile`** — системный файл, выполняется для всех пользователей при login-входе. Обычно не трогают вручную.
- **`~/.bash_profile`** — твой личный файл для login shell. Здесь обычно устанавливают переменные окружения (`PATH`, `JAVA_HOME` и т.д.), которые нужны один раз при входе. Хорошая практика — в конце написать `source ~/.bashrc`, чтобы не дублировать настройки.
- **`~/.bash_login`** и **`~/.profile`** — резервные варианты: Bash ищет их только если не нашёл `.bash_profile`. `.profile` также читается другими оболочками (sh, dash), поэтому он есть во многих системах по умолчанию.
- **`~/.bashrc`** — самый важный для повседневного использования. Запускается при каждом открытии нового терминала. Сюда кладут: псевдонимы (`alias ll='ls -la'`), настройку prompt (`PS1` [[#Сноска|link]]), функции, подключение менеджеров версий (nvm, pyenv).
- **`~/.bash_logout`** — запускается при выходе из login shell. Используется редко (очистка временных файлов, сообщение "пока").
- **`~/.bash_aliases`** — не стандартный, но популярный: просто отдельный файл для псевдонимов, который подключают из `.bashrc` командой `source ~/.bash_aliases`.

**Практический совет**: большинство личных настроек пиши в `~/.bashrc`. В `~/.bash_profile` достаточно одной строки `source ~/.bashrc`.

# Сноска
**`PS1`** (Prompt String 1) — это переменная, которая определяет как выглядит строка приглашения в терминале. То, что ты видишь перед каждой командой.

По умолчанию это что-то скучное вроде:

```
user@hostname:~$
```
