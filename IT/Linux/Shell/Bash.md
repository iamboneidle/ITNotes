# 1. Разница объявления переменной с `export` и без
В чем разница м/у `export VAR="VALUE"` и `VAR="VALUE"`:

При объявлении переменной через **export** - переменная будет доступна в любых других процессах. При обычном объявлении переменной - переменная будет доступна только в запущенном процессе.
___
# 2.  Что значит `$@`, `$!`, `$?`, `$$` в bash?
- `$@` - показывает все параметры переданные скрипту.
- `$!` - показывает pid последнего процесса, которая оболочка запустила в фоновом режиме.
- `$$` - показывает текущий pid процесса.
- `$?` - показывает с каким кодом завершилась последняя выполненная функция. 0 - успешное выполнение.
___
# 3. Фильтрация STDOUT & STDERR
```bash
cmd 2>&1 >/dev/null | grep pattern
```
1. **Создается pipe** → stdout `cmd` направляется в pipe
2. **`2>&1`** → stderr копирует текущий stdout (pipe) → stderr тоже в pipe
3. **`>/dev/null`** → stdout перенаправляется в `/dev/null` (перезаписывает pipe для stdout)
4. **Итог**: stdout → `/dev/null` (пропадает), stderr → pipe → `grep`

# 4. Типы переменных
## 4.1. Строковые переменные (по умолчанию)
```bash
name="John"
text="Hello, World!"
empty_var=""
```
## 4.2. Целочисленные переменные
Нужно явно объявить с помощью `declare -i` или использовать арифметическое окружение `$(( ))`:
```bash
declare -i count=10
count=count+5          # работает как арифметика
echo $count            # 15
# Без declare -i:
num=5
num=num+2              # будет строка "5+2"
echo $((num+2))        # 7 (арифметическое расширение)
```
## 4.3. Массивы
```bash
arr=("one" "two" "three")
echo ${arr[0]}         # one
arr[3]="four"
echo ${#arr[@]}        # длина массива
```
## 4.4. Ассоциативные массивы (э4.ши)
Требуют `declare -A` (bash 4+):
```bash
declare -A user
user["name"]="Alice"
user["age"]=30
echo ${user["name"]}   # Alice
```
## 4.5. Переменные окружения (экспортируемые)
```bash
export PATH="/custom/bin:$PATH"
export MY_VAR="value"
```
## 4.6. Только для чтения
```bash
readonly PI=3.14
PI=3.1415              # ошибка: readonly variable
```
## 4.7. Строковые переменные с флагами (declare)
```bash
declare -r ROOT="/var"     # readonly
declare -l lower="HELLO"   # автоматически в нижний регистр
declare -u upper="world"   # автоматически в верхний регистр
```
## 4.8. Арифметика без объявления типа
```bash
#Использование $(( ))
result=$((10 + 20))
echo $result            # 30
```
```bash
# let команда
let "x = 5 * 3"
echo $x                 # 15
```

## 4.9. Проверка типа (атрибута):
```bash
declare -p var_name     # показывает атрибуты переменной
```
## 4.10. Особенность:
Всё, что не объявлено как `-i` или не используется в `$(( ))`, **трактуется как строка**. Даже если вы запишете `var=123`, это будет строка "123", просто при необходимости bash может интерпретировать её как число.
