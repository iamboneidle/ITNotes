# 1. Что такое AWK
`awk` — это интерпретируемый язык для построчной обработки текстовых данных. Назван по первым буквам фамилий создателей: **A**ho, **W**einberger, **K**ernighan (Bell Labs, 1977).

AWK читает входной поток **строка за строкой**, разбивает каждую строку на **поля**, и применяет к ней набор правил `паттерн { действие }`.

Основные реализации:
- `awk` — обычно ссылается на `mawk` или `gawk` в зависимости от дистрибутива
- `gawk` — GNU AWK, самая распространённая, с расширениями
- `mawk` — быстрая минималистичная реализация
- `nawk` — «новый» awk, стандартный на BSD/macOS
Проверить версию:
```bash
awk --version
gawk --version
```

# 2. Базовый синтаксис
## 2.1. Структура программы
Программа AWK состоит из правил вида:
```
паттерн { действие }
паттерн { действие }
...
```
- **Паттерн** — условие, при котором выполняется действие (может отсутствовать — тогда действие выполняется для каждой строки).
- **Действие** — блок кода в фигурных скобках (может отсутствовать — тогда строка просто печатается).
Примеры:
```bash
# Нет паттерна — действие для каждой строки
awk '{ print $1 }' file.txt

# Нет действия — строки, совпавшие с паттерном, печатаются целиком
awk '/error/' file.txt

# Оба присутствуют
awk '/error/ { print $2, $3 }' file.txt
```
## 2.2. Запуск AWK
```bash
# Из аргумента командной строки
awk 'программа' файл1 файл2

# Из файла
awk -f script.awk файл.txt

# Из stdin через pipe
cat файл.txt | awk '{ print $1 }'

# Задать разделитель полей
awk -F ':' '{ print $1 }' /etc/passwd

# Передать переменную извне
awk -v threshold=100 '$2 > threshold { print }' data.txt
```

# 3. Поля и разделители
## 3.1. Встроенные переменные полей
AWK автоматически разбивает каждую строку на поля по разделителю (по умолчанию — любой пробельный символ).

|Переменная|Значение|
|---|---|
|`$0`|Вся строка целиком|
|`$1`|Первое поле|
|`$2`|Второе поле|
|`$NF`|Последнее поле (NF = число полей)|
|`$(NF-1)`|Предпоследнее поле|

Пример с `/etc/passwd`:
```bash
awk -F ':' '{ print $1, $NF }' /etc/passwd
# Вывод: имя_пользователя оболочка
```
Изменение поля меняет `$0`:
```bash
echo "hello world" | awk '{ $2 = "AWK"; print $0 }'
# Вывод: hello AWK
```
## 3.2. Изменение разделителя
```bash
# Входной разделитель — FS (Field Separator)
awk -F ',' '{ print $1 }' file.csv          # через флаг
awk 'BEGIN { FS="," } { print $1 }' file.csv # через переменную

# Множество разделителей (регулярка)
awk -F '[,;:]' '{ print $2 }' file.txt

# Выходной разделитель — OFS (Output Field Separator)
awk -F ':' 'BEGIN { OFS="\t" } { print $1, $3 }' /etc/passwd

# Выходной разделитель строк — ORS (Output Record Separator)
awk 'BEGIN { ORS=", " } { print $1 }' file.txt
```

# 4. Паттерны (условия)
## 4.1. BEGIN и END
Специальные паттерны, выполняющиеся один раз — до и после обработки файла.
```bash
awk '
BEGIN {
    print "=== Начало обработки ==="
    count = 0
}
{
    count++
}
END {
    print "=== Обработано строк:", count, "==="
}' file.txt
```
`BEGIN` часто используют для инициализации переменных и задания `FS`/`OFS`.
## 4.2. Регулярные выражения
```bash
# Строки, содержащие "error"
awk '/error/ { print }' log.txt

# Строки, НЕ содержащие "error"
awk '!/error/ { print }' log.txt

# Паттерн на конкретное поле (~  и !~)
awk '$3 ~ /^[0-9]+$/ { print $1, $3 }' data.txt
awk '$3 !~ /^[0-9]+$/ { print "Не число:", $3 }' data.txt
```
## 4.3. Диапазоны
Диапазон `паттерн1, паттерн2` — выводит строки от первого совпадения до второго (включительно).
```bash
# Всё между "START" и "END"
awk '/START/, /END/ { print }' file.txt

# Строки с 5 по 10
awk 'NR==5, NR==10 { print }' file.txt
```

# 5. Встроенные переменные

|Переменная|Описание|Пример значения|
|---|---|---|
|`FS`|Разделитель входных полей|`" "` (по умолч.)|
|`OFS`|Разделитель выходных полей|`" "` (по умолч.)|
|`RS`|Разделитель входных записей|`"\n"` (по умолч.)|
|`ORS`|Разделитель выходных записей|`"\n"` (по умолч.)|
|`NR`|Номер текущей записи (строки) от начала обработки|`1`, `2`, `3`, ...|
|`NF`|Количество полей в текущей строке|`5`|
|`FNR`|Номер строки в текущем файле (сбрасывается для нового файла)|`1`, `2`, ...|
|`FILENAME`|Имя текущего обрабатываемого файла|`data.txt`|
|`ARGC`|Число аргументов командной строки|`2`|
|`ARGV`|Массив аргументов командной строки|`ARGV[0]="awk"`|
|`ENVIRON`|Массив переменных окружения (gawk)|`ENVIRON["HOME"]`|

Примеры использования:
```bash
# Номер строки перед каждой строкой
awk '{ print NR": "$0 }' file.txt

# Пропустить первую строку (заголовок)
awk 'NR > 1 { print }' data.csv

# Вывести только непустые строки
awk 'NF > 0 { print }' file.txt

# FNR при обработке нескольких файлов
awk 'FNR == 1 { print "Файл:", FILENAME }' file1.txt file2.txt
```

# 6. Операторы и выражения
## 6.1. Арифметика
AWK поддерживает стандартные арифметические операции и работает с числами с плавающей точкой.

|Оператор|Описание|
|---|---|
|`+`|Сложение|
|`-`|Вычитание|
|`*`|Умножение|
|`/`|Деление|
|`%`|Остаток|
|`^` или `**`|Степень|
|`++`, `--`|Инкремент/декремент|
|`+=`, `-=`, `*=`, `/=`|Присваивание с операцией|

```bash
# Суммировать второй столбец
awk '{ sum += $2 } END { print "Сумма:", sum }' data.txt

# Среднее значение
awk '{ sum += $1; count++ } END { print sum/count }' data.txt
```
## 6.2. Строковые операции
AWK различает числа и строки по контексту. Конкатенация строк — просто пробел между значениями.
```bash
# Конкатенация
awk '{ full = $1 " " $2; print full }' names.txt

# Сравнение строк
awk '$1 == "root" { print "Root найден" }' /etc/passwd
```
## 6.3. Сравнения

|Оператор|Описание|
|---|---|
|`==`|Равно|
|`!=`|Не равно|
|`<`, `>`|Меньше, больше|
|`<=`, `>=`|Меньше или равно, больше или равно|
|`~`|Совпадает с регулярным выражением|
|`!~`|Не совпадает с регулярным выражением|

Логические операторы: `&&` (И), `||` (ИЛИ), `!` (НЕ).
```bash
# Несколько условий
awk '$1 > 100 && $2 ~ /^A/ { print }' data.txt

# Тернарный оператор
awk '{ status = ($3 > 0) ? "положит." : "отрицат."; print $1, status }' data.txt
```

# 7. Управляющие конструкции
## 7.1. if / else
```bash
awk '{
    if ($2 > 1000) {
        print $1, "высокий:", $2
    } else if ($2 > 500) {
        print $1, "средний:", $2
    } else {
        print $1, "низкий:", $2
    }
}' data.txt
```
## 7.2. while и for
```bash
# while
awk '{
    i = 1
    while (i <= NF) {
        print "Поле " i ":", $i
        i++
    }
}' data.txt

# for (C-style)
awk '{
    for (i = 1; i <= NF; i++) {
        printf "%-10s", $i
    }
    print ""
}' data.txt

# for-in для массивов
awk '{
    arr[$1] += $2
}
END {
    for (key in arr) {
        print key, arr[key]
    }
}' data.txt
```
## 7.3. next и exit
```bash
# next — перейти к следующей строке (пропустить остаток блока)
awk '{
    if ($1 == "#") next   # пропустить строки-комментарии
    print $0
}' file.txt

# exit — завершить обработку (блок END всё равно выполнится)
awk 'NR == 5 { print "Найдено на строке 5!"; exit } { print }' file.txt
```

# 8. Массивы
## 8.1. Ассоциативные массивы
В AWK все массивы **ассоциативные** (словари). Ключ — всегда строка, но можно использовать числа (они автоматически приводятся к строке).
```bash
# Подсчёт встречаемости значений
awk '{ count[$1]++ }
END {
    for (word in count)
        print count[word], word
}' file.txt | sort -rn

# Проверить наличие ключа
awk '{
    if ($1 in seen) {
        print "Дубликат:", $1
    } else {
        seen[$1] = 1
    }
}' file.txt

# Многомерный (составной) ключ
awk '{ data[$1","$2] += $3 }
END {
    for (k in data) print k, data[k]
}' file.txt
```
## 8.2. Удаление элементов
```bash
# Удалить один элемент
awk '{ arr[$1]=$2 }
END {
    delete arr["unwanted"]
    for (k in arr) print k, arr[k]
}' data.txt

# Удалить весь массив (gawk)
awk 'END { delete arr }' data.txt
```

# 9. Встроенные функции
## 9.1. Строковые функции

|Функция|Описание|Пример|
|---|---|---|
|`length(s)`|Длина строки или массива|`length($0)`|
|`substr(s, m, n)`|Подстрока с позиции m длиной n|`substr($1, 1, 3)`|
|`index(s, t)`|Позиция подстроки t в s (0 если нет)|`index($0, "err")`|
|`split(s, a, fs)`|Разбить строку на массив|`split($0, a, ",")`|
|`sub(r, s, t)`|Заменить первое вхождение r на s в t|`sub(/foo/, "bar", $0)`|
|`gsub(r, s, t)`|Заменить все вхождения r на s в t|`gsub(/\s+/, "_", $1)`|
|`match(s, r)`|Найти r в s, установить RSTART/RLENGTH|`match($0, /[0-9]+/)`|
|`toupper(s)`|В верхний регистр|`toupper($1)`|
|`tolower(s)`|В нижний регистр|`tolower($1)`|
|`sprintf(fmt, ...)`|Форматированная строка|`sprintf("%05d", $1)`|

Примеры:
```bash
# Извлечь первые 8 символов
awk '{ print substr($1, 1, 8) }' file.txt

# Заменить все пробелы на подчёркивание
awk '{ gsub(/ /, "_"); print }' file.txt

# Разбить по запятой и напечатать третий элемент
awk '{ n = split($0, a, ","); print a[3] }' file.csv

# Найти число в строке
awk 'match($0, /[0-9]+/) { print substr($0, RSTART, RLENGTH) }' file.txt
```
## 9.2. Математические функции

|Функция|Описание|
|---|---|
|`sin(x)`, `cos(x)`, `atan2(y,x)`|Тригонометрия (в радианах)|
|`exp(x)`|Экспонента e^x|
|`log(x)`|Натуральный логарифм|
|`sqrt(x)`|Квадратный корень|
|`int(x)`|Целая часть (обрезание, не округление)|
|`rand()`|Случайное число [0, 1)|
|`srand(x)`|Инициализировать генератор случайных чисел|

```bash
# Округлить до 2 знаков
awk '{ printf "%.2f\n", $1 }' data.txt

# Случайная выборка ~10% строк
awk 'rand() < 0.1 { print }' file.txt
```
## 9.3. Ввод/вывод
```bash
# print — вывод с ORS в конце (обычно \n)
awk '{ print $1, $2 }' file.txt

# printf — форматированный вывод без автоматического \n
awk '{ printf "%-20s %5d\n", $1, $2 }' file.txt

# Вывод в файл
awk '{ print $0 > "output.txt" }' file.txt

# Добавление в файл
awk '{ print $0 >> "log.txt" }' file.txt

# Вывод в команду через pipe
awk '{ print $1 | "sort -u" }' file.txt

# Чтение из команды
awk 'BEGIN { "date" | getline today; print today }'

# getline — прочитать следующую строку
awk '/HEADER/ { getline; print "После заголовка:", $0 }' file.txt
```

# 10. Пользовательские функции
AWK поддерживает пользовательские функции. Локальные переменные объявляются как дополнительные параметры с пробелами.
```bash
awk '
function max(a, b) {
    return (a > b) ? a : b
}

function trim(s,    # локальные переменные — через дополнительные параметры:
             result) {
    result = s
    gsub(/^[ \t]+|[ \t]+$/, "", result)
    return result
}

{
    print max($1, $2), trim($3)
}' data.txt
```
```bash
# Вычислить факториал
awk '
function factorial(n) {
    if (n <= 1) return 1
    return n * factorial(n - 1)
}
BEGIN {
    for (i = 1; i <= 10; i++)
        print i"! =", factorial(i)
}'
```

# 11. Практические примеры
## 11.1. Подсчёт и агрегация
```bash
# Сумма столбца
awk '{ sum += $1 } END { print sum }' data.txt

# Максимум и минимум
awk 'BEGIN { max = -999999; min = 999999 }
     { if ($1 > max) max = $1; if ($1 < min) min = $1 }
     END { print "max:", max, "min:", min }' data.txt

# Количество строк, слов, символов (аналог wc)
awk '{ lines++; words += NF; chars += length($0) + 1 }
END { print lines, words, chars }' file.txt

# Частота значений с сортировкой
awk '{ freq[$1]++ }
END { for (k in freq) print freq[k], k }' file.txt | sort -rn
```
## 11.2. Фильтрация строк
```bash
# Строки, где второй столбец > 100
awk '$2 > 100' data.txt

# Строки между двумя паттернами
awk '/BEGIN_MARKER/, /END_MARKER/' file.txt

# Уникальные строки (как uniq, но без сортировки)
awk '!seen[$0]++' file.txt

# Каждая N-ая строка (например, каждая третья)
awk 'NR % 3 == 0' file.txt

# Строки с нужным числом полей
awk 'NF == 5' file.txt
```
## 11.3. Обработка CSV
```bash
# Простой CSV (без экранирования кавычек)
awk -F ',' '{ print $1, $3 }' data.csv

# Поменять порядок столбцов
awk -F ',' 'BEGIN { OFS="," } { print $3, $1, $2 }' data.csv

# Пропустить заголовок
awk -F ',' 'NR > 1 { print $1, $2 }' data.csv

# Добавить вычисляемый столбец
awk -F ',' 'BEGIN { OFS="," }
NR == 1 { print $0, "total" }
NR > 1  { print $0, $2 + $3 }' data.csv
```
## 11.4. Работа с логами
```bash
# Подсчёт ошибок по типу
awk '/ERROR/ { match($0, /ERROR: ([^,]+)/, a); errors[a[1]]++ }
END { for (e in errors) print errors[e], e }' app.log | sort -rn

# Найти IP с наибольшим числом запросов (nginx access log)
awk '{ ip[$1]++ }
END { for (k in ip) print ip[k], k }' access.log | sort -rn | head -10

# Извлечь строки за определённое время
awk '$4 >= "[10/Apr/2025:14:00" && $4 <= "[10/Apr/2025:15:00"' access.log

# Подсчёт HTTP кодов ответа
awk '{ codes[$9]++ }
END { for (c in codes) print c, codes[c] }' access.log | sort -k1n
```

# 12. AWK vs sed vs grep

|Задача|Лучший инструмент|
|---|---|
|Найти строки по паттерну|`grep`|
|Простые замены в потоке|`sed`|
|Работа с полями/столбцами|`awk`|
|Агрегация, подсчёт, суммы|`awk`|
|Сложная логика и условия|`awk`|
|Многострочные паттерны|`awk` или `sed`|

Типичная схема:
```bash
# grep для фильтрации → awk для обработки → sort для сортировки
grep "ERROR" app.log | awk '{ print $4, $NF }' | sort | uniq -c | sort -rn
```
**Когда AWK не подходит:**
- Бинарные файлы
- Многострочные JSON/XML (лучше `jq`, `xmllint`)
- Сложная логика с большими данными (лучше Python/Perl)
