# 1. Определение
**IPVS** — это встроенный в ядро Linux балансировщик нагрузки уровня 4 (TCP/UDP). Он позволяет объединить несколько реальных серверов (Real Servers) в один виртуальный сервис с единым IP-адресом (VIP). В отличие от `iptables`, который использует цепочки правил (линейный поиск), IPVS использует хэш-таблицы, что делает его значительно быстрее при работе с тысячами сервисов.
___
# 2. Как работает IPVS
IPVS перехватывает пакеты на этапе `LOCAL_IN` (когда пакет пришел на сетевой интерфейс и предназначен самому хосту) и, если пакет соответствует заданному виртуальному сервису, перенаправляет его на один из реальных серверов.

**Основные компоненты:**
- **VIP (Virtual IP)**: публичный адрес, на который обращаются клиенты.
- **DIP (Director IP)**: адрес балансировщика в локальной сети.
- **RIP (Real IP)**: адреса серверов, которые реально обрабатывают запросы.
___
# 3. Базовые команды ipvsadm
Для управления IPVS используется утилита `ipvsadm`.

- Просмотр таблицы балансировки: `ipvsadm -L -n`
- Создание виртуального сервиса: `ipvsadm -A -t 192.168.1.10:80 -s rr` _(Здесь `-A` — добавить сервис, `-t` — TCP, `-s rr` — алгоритм Round Robin)_.
- Добавление реального сервера в сервис: `ipvsadm -a -t 192.168.1.10:80 -r 10.0.0.5:80 -g` (Здесь `-a` — добавить сервер, `-r` — адрес сервера, `-g` — метод пересылки Gateway/DR).
- Удаление правила: `ipvsadm -D -t 192.168.1.10:80`
- Очистка всех правил: `ipvsadm -C`
___
# 4. Алгоритмы планирования
IPVS поддерживает множество стратегий распределения трафика:
1. **rr** (Round Robin): по очереди на каждый сервер.
2. **wrr** (Weighted Round Robin): с учетом веса (мощности) сервера.
3. **lc** (Least Connection): на сервер с наименьшим числом активных соединений.
4. **wlc** (Weighted Least Connection): учитывает и вес, и количество соединений (самый популярный).
5. **sh** (Source Hashing): привязывает клиента к конкретному серверу по его IP.
___
# 5. Методы пересылки пакетов
IPVS может "доставлять" пакет до сервера тремя способами:

- **NAT (Masquerading)**: Балансировщик меняет адрес назначения в пакете. Ответ сервера также идет через балансировщик.
- **DR (Direct Routing)**: Пакет передается серверу на уровне L2 (MAC-адрес). Ответ сервера идет напрямую клиенту в обход балансировщика. Самый быстрый метод.
- **Tunneling (IPIP)**: Пакет инкапсулируется в другой IP-пакет. Позволяет серверам находиться в разных сетях.
___
# 6. Пример настройки
Создадим балансировщик для двух веб-серверов:
```shell
# 1. Добавляем виртуальный IP (VIP)
ip addr add 192.168.50.100/32 dev eth0

# 2. Создаем TCP сервис на порту 80 с алгоритмом "взвешенная очередь"
ipvsadm -A -t 192.168.50.100:80 -s wlc

# 3. Добавляем два реальных сервера (метод NAT)
ipvsadm -a -t 192.168.50.100:80 -r 192.168.1.10:80 -m -w 1
ipvsadm -a -t 192.168.50.100:80 -r 192.168.1.11:80 -m -w 2

# 4. Включаем форвардинг в ядре (обязательно для NAT)
sysctl -w net.ipv4.ip_forward=1
```
