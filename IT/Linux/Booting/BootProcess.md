# 1. Краткий список шагов
1. **Power On** — питание подаётся на материнскую плату
2. **BIOS / UEFI** — CPU переходит на адрес прошивки, запускается самотестирование (POST)
3. **Поиск загрузчика** — BIOS/UEFI ищет MBR или GPT на диске
4. **Bootloader Stage 1** — 446 байт кода из MBR запускают Stage 2
5. **Bootloader Stage 2** — (GRUB и др.) загружает ядро и `initrd`
6. **Ядро Linux** — инициализирует CPU, память, устройства
7. **init / systemd** — первый процесс пользовательского пространства
8. **Сервисы** — запускаются все юниты/демоны
9. **Login** — появляется приглашение входа 10. **Shell** — выполняются `.bashrc`, `.zshrc`, `.zprofile` и др.

# 2. Подробное описание
## 2.1 Power On
При нажатии кнопки питания напряжение стабилизируется и материнская плата подаёт сигнал `POWER_GOOD`. После этого CPU аппаратно переходит на фиксированный адрес в памяти — там находится точка входа BIOS/UEFI.
## 2.2 BIOS / UEFI — POST
**POST (Power-On Self-Test)** — набор проверок железа:
- проверка оперативной памяти
- проверка CPU, шин, контроллеров
- инициализация базового I/O

> **BIOS** — legacy, 16-битный реальный режим, ограничен MBR.  
> **UEFI** — современный стандарт, поддерживает GPT, Secure Boot, драйверы на этапе загрузки.
## 2.3 Поиск загрузчика: MBR vs GPT

|###############|MBR|GPT|
|---|---|---|
|Расположение|Первые 512 байт диска|Весь диск|
|Макс. размер диска|2 ТБ|8 ЗБ (8 млрд. ТБ)|
|Кол-во разделов|до 4 первичных|до 128|
|Совместимость|BIOS|UEFI (BIOS ч/з гибрид)|
**MBR структура:**
- 446 байт — Stage 1 загрузчик
- 64 байта — таблица разделов
- 2 байта — сигнатура `0x55AA`
## 2.4 Bootloader Stage 1
446 байт — слишком мало для полноценного загрузчика. Поэтому Stage 1 делает одно: **находит и передаёт управление Stage 2**.  
При UEFI Stage 1 фактически пропускается — прошивка сама умеет читать файловую систему EFI-раздела (`ESP`) и запускать `.efi`-файл напрямую.
## 2.5 Bootloader Stage 2 (GRUB2 и др.)
Типичные загрузчики: **GRUB2**, systemd-boot, rEFInd.  
Stage 2 умеет:
- читать файловые системы (ext4, btrfs, xfs...)
- показывать меню выбора ОС
- передавать параметры ядру (`ro quiet splash`)
- загружать **ядро** (`vmlinuz`) и **initial RAM disk** (`initrd` / `initramfs`)  
    **initramfs** — временная корневая файловая система в RAM. Содержит минимальный набор модулей ядра (драйверов), необходимых для монтирования настоящего корневого раздела.
## 2.6 Ядро Linux После загрузки ядро:
1. Распаковывает себя в памяти
2. Инициализирует подсистемы: память (MMU), планировщик, прерывания
3. Монтирует `initramfs` как временный `/`
4. Загружает необходимые модули (драйверы дисков, файловых систем)
5. Монтирует настоящий корневой раздел (`/`)
6. Запускает первый процесс пользовательского пространства
## 2.7 init / systemd
Первый процесс — всегда **PID 1**.  
Исторически это был `/sbin/init` (SysVinit). Сейчас в большинстве дистрибутивов — **systemd**.  
systemd читает юниты (`*.service`, `*.target`) и строит граф зависимостей для параллельного запуска сервисов.
## 2.8 Запуск сервисов
systemd поднимает сервисы согласно **target**'ам:
- `sysinit.target` → базовая инициализация
- `basic.target` → сеть, логирование
- `multi-user.target` → все фоновые сервисы
- `graphical.target` → дисплейный менеджер (если есть GUI)
## 2.9 Login
- Запускается `getty` (или дисплейный менеджер — GDM, SDDM и др.).
- Пользователь вводит логин/пароль → PAM проверяет аутентификацию → создаётся пользовательская сессия.
