# 1. Что такое псевдофайловая система
**Псевдофайловая система** означает файловую систему, которая не имеет _фактических_ файлов - скорее, она имеет виртуальные записи, которые сама файловая система создает на месте.
Можно сказать это интерфейс ядра linux.

Например, `/proc` во многих ОС - это `procfs`, который динамически генерирует каталоги для каждого процесса.  
Точно так же `/sys` в Linux генерирует файлы и каталоги для представления аппаратных схем. Есть FUSE на основе псевдо-файловая система для _многих_ вещей.

`/dev` может быть реальной файловой системой (просто подкаталогом `/`) или виртуальной псевдофайловой системой (например, `devfs`), или средней точкой, такой как Linux `devtmpfs` (которая является полной файловой системой в памяти, но все же создает узлы устройства из нигде).

Если говорить о том, где хранится `proc`, то это **представление** структуры ядра. И можно сказать, что содержимое хранится в памяти ядра, то есть в ОЗУ.

# 2. Основные
| Имя          | Типовая точка монтирования | Краткое описание и назначение                                                                                                                                                                               |
| ------------ | -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **procfs**   | `/proc`                    | **Информация о процессах и ядре.** Самая известная псевдо-ФС. Содержит каталоги для каждого процесса (по PID) и файлы с системной информацией (`cpuinfo`, `meminfo` и др.).                                 |
| **sysfs**    | `/sys`                     | **Информация об устройствах и драйверах.** Предоставляет детальную информацию об устройствах, модулях ядра и их параметрах. Это современная замена многих функций `procfs` для управления оборудованием.    |
| **tmpfs**    | `/dev/shm`, `/tmp`         | **Временное хранилище в оперативной памяти.** Позволяет создавать файлы и каталоги, которые физически находятся в оперативной памяти (или swap-разделе). Очень быстро, но данные теряются при перезагрузке. |
| **devtmpfs** | `/dev`                     | **Управление файлами устройств.** Автоматически создает и управляет файлами устройств (например, `/dev/sda`), упрощая работу с периферией.                                                                  |
| **devpts**   | `/dev/pts`                 | **Псевдо-терминалы.** Обеспечивает работу эмуляторов терминалов (например, `xterm`, `ssh`), создавая для каждого из них специальный файл.                                                                   |
| **cgroupfs** | `/sys/fs/cgroup`           | **Контроль ресурсов (Control Groups).** Используется для ограничения и учета потребления ресурсов (CPU, память, дисковый ввод-вывод) группами процессов.                                                    |
