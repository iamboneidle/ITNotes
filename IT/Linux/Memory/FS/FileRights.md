# 1. Базовое понятие "Три слоя"
Для любого файла или папки Linux ([[Inode]]) помнит три группы пользователей:
1. **Владелец (User / u)** — кто создал (или кому назначили).
2. **Группа (Group / g)** — набор пользователей (например, `developers`, `admins`).
3. **Остальные (Others / o)** — все остальные на системе.

И для каждой группы есть три действия:

|Буква|Право|Для файла|Для папки|
|---|---|---|---|
|**r** (read)|Чтение|Посмотреть содержимое файла|Посмотреть **список** файлов в папке (`ls`)|
|**w** (write)|Запись|Изменить файл|**Создавать / удалять** файлы в папке|
|**x** (execute)|Выполнение|**Запустить** как программу/скрипт|**Зайти внутрь** (`cd`)|
___
# 2. Вывод команды `ls`
```bash
-rwxr-xr-- 1 user group 1234 Mar 3 10:00 myfile.sh
```
Этот вывод расшифровывается по кусочкам:

|Позиция|Символ|Что значит|
|---|---|---|
|0 (самый первый)|`-`|Тип: `-` файл, `d` папка, `l` симлинк|
|1-3|`rwx`|Права **владельца** (user)|
|4-6|`r-x`|Права **группы** (group)|
|7-9|`r--`|Права **остальных** (others)|
В примере:
- **Владелец** может `r` читать, `w` писать, `x` запускать.
- **Группа** может `r` читать, `x` запускать (но не менять).
- **Остальные** могут `r` только читать.
___
# 3. Восьмеричный пример
Каждое право имеет "вес" в битах:

|Право|Значение|
|---|---|
|`r`|4|
|`w`|2|
|`x`|1|
Чтобы получить цифру для одной группы, складываем нужные права.
**Примеры:**

|Двоичное|Сумма|Права|
|---|---|---|
|`rwx`|4+2+1 = **7**|Читать, писать, выполнять|
|`rw-`|4+2+0 = **6**|Читать, писать (но не выполнять)|
|`r-x`|4+0+1 = **5**|Читать, выполнять (но не писать)|
|`r--`|4+0+0 = **4**|Только читать|
|`---`|0+0+0 = **0**|Нет прав|
**Полная запись — три цифры подряд (владелец, группа, остальные):**
- `644` → владелец `6` (`rw-`), группа `4` (`r--`), остальные `4` (`r--`) — классика для текстовых файлов.
- `755` → владелец `7` (`rwx`), группа `5` (`r-x`), остальные `5` (`r-x`) — классика для скриптов и папок.
- `777` → **все могут всё** — почти всегда плохая идея.
- `600` → только владелец читает и пишет, никто больше ничего — для секретов (ключи SSH).
---
# 4. Почему *0*644?
**Ответ**: потому что восьмеричная система счисления.

`0777` — это **восьмеричная запись** прав доступа. Без нуля число `777` — десятичное, с нулем — восьмеричное. В терминале `chmod 777` угадает ваш замысел, но в коде всегда пишите `0777` (или `0o777` в Python 3), чтобы не получить случайно `chmod 1411`.

# 5. `chmod` & `chown`
 `chmod` — изменить права:
 ```shell
 # Символьный способ (понятный)
chmod u+x script.sh      # владельцу (+x) добавить выполнение
chmod go-w file.txt      # группе и остальным убрать запись (-w)
chmod a=r file.txt       # всем (all) только чтение

# Цифровой способ (быстрый)
chmod 755 script.sh      # rwxr-xr-x
chmod 644 README.md      # rw-r--r--
chmod 600 ~/.ssh/id_rsa  # только владелец читает/пишет
 ```
`chown` — сменить владельца и группу:
```shell
chown user:group file.txt   # новый владелец и группа
chown user file.txt         # только владельца
chown :group file.txt       # только группу
chown -R user:group folder/ # рекурсивно для всей папки
```

# 6. Специальные биты (Sticky, SUID, SGID)
Иногда вы видите `chmod 1755` или букву `t` в конце (`drwxrwxrwt`).

| Бит               | Число | Эффект                                                                                                                              |
| ----------------- | ----- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **SUID** (setuid) | 4     | При запуске программа выполняется от **владельца** файла, не от того, кто запустил. Пример: `passwd` (меняет пароль, требует root). |
| **SGID** (setgid) | 2     | Для файлов — как SUID, но для группы. Для папок — новые файлы наследуют группу папки.                                               |
| **Sticky**        | 1     | В папке с sticky (`/tmp`) удалить файл может только его владелец (даже если у остальных есть запись).                               |
Задаются четвертой цифрой:
```shell
chmod 4755 file   # SUID + rwsr-xr-x
chmod 2770 folder # SGID + rwxrws---
chmod 1777 /tmp   # Sticky + rwxrwxrwt
```
В `ls -l` вы увидите:
- `rws` вместо `rwx` (SUID)
- `r-s` вместо `r-x` (SGID)
- `rwt` вместо `rwx` (Sticky)
