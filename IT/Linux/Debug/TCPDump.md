# 1. Базовый синтаксис
Используй флаги-ограничители:
- `-i eth0`: конкретный интерфейс (или `-i any` для всех, но осторожно с нагрузкой).
- `-n`: не резолвить IP в имена (DNS-запросы в процессе дебага — это лишний шум и задержки).
- `-nn`: не резолвить ни имена, ни порты (покажет `80` вместо `http`).
- `-w file.pcap`: запись в файл для последующего анализа в Wireshark.
___
# 2. Фильтрация (BPF - Berkeley Packet Filter)
**По направлению и портам:**
- `host 10.244.1.5`: всё, что идет от этого IP или к нему.
- `net 192.168.1.0/24`: трафик целой подсети.
- `port 80`: только HTTP.
- `portrange 1-1024`: системные порты.

**Комбинирование (Логика):**
- `src host 1.1.1.1 and dst port 443`: только исходящий HTTPS трафик к конкретному хосту.
- `tcp[tcpflags] & (tcp-syn|tcp-ack) == tcp-syn`: фильтр только для **SYN-пакетов** (поиск попыток установить соединение).
