# 0. Вводная
**Межпроцессное взаимодействие (IPC — Inter-Process Communication)** — это набор механизмов, которые позволяют разным процессам обмениваться данными и синхронизировать свою работу.

В Linux процессы изолированы друг от друга (у каждого свое адресное пространство). Чтобы один процесс мог «дотянуться» до другого, нужно использовать средства ядра.
___
# 1. Сигналы (Signals)
[[Signals]]
Самый примитивный и быстрый способ. Это уведомление процесса о событии.
- **Как работает:** Ядро прерывает выполнение процесса и заставляет его выполнить обработчик (handler) сигнала.
- **Ограничение:** Нельзя передать данные (кроме самого факта сигнала).
- **Примеры:** `SIGKILL` (9), `SIGTERM` (15), `SIGHUP` (перечитать конфиг).
___
# 2. Каналы (Pipes)
Классика UNIX, которую мы видим в консоли: `ls | grep`.
**Anonymous Pipes (Анонимные каналы)**
- **Суть:** Однонаправленный поток данных. Работает только между родственными процессами (parent-child).
- **Механика:** Данные хранятся в буфере ядра. Если буфер полон, пишущий процесс блокируется.
**Named Pipes (FIFO)**
- **Суть:** Канал, представленный в виде файла в файловой системе (`mkfifo /tmp/mypipe`).
- **Плюс:** Могут общаться совершенно не связанные процессы.
___
# 3. Разделяемая память (Shared Memory)
**Самый быстрый IPC**
- **Механика:** Два или более процесса отображают одну и ту же физическую область RAM в свои виртуальные адресные пространства.
- **Почему быстро:** Данные не копируются между процессом и ядром (в отличие от пайпов или сокетов). Процесс просто пишет в память, а другой тут же это видит.
- **Проблема:** Нужна синхронизация (мьютексы/семафоры), иначе процессы перезапишут данные друг друга.
___
# 4. Очереди сообщений (Message Queues)
- **Суть:** Список сообщений, хранящийся в ядре.
- **Плюс:** В отличие от пайпов (где данные — это поток байтов), здесь данные передаются дискретными блоками (сообщениями). Процесс может прочитать сообщение определенного типа, пропустив остальные.
___
# 5. Сокеты (Sockets)
[[Sockets]]
**Unix Domain Sockets (UDS)**
- **Суть:** Обмен данными через файл-сокет на диске (например, `/var/run/docker.sock`).
- **Для кого:** Только для процессов внутри одного хоста.
- **Производительность:** Быстрее, чем сетевые TCP-сокеты, так как нет оверхеда на инкапсуляцию в пакеты, расчет контрольных сумм и прохождение сетевого стека.

**Network Sockets (TCP/IP, UDP)**
- **Суть:** Обмен данными через сетевой стек.
- **Плюс:** Процессы могут находиться на разных серверах.
